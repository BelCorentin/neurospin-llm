====== Data Storage at NeuroSpin ======

This page describes the data storage infrastructure at NeuroSpin, including paths, quotas, naming conventions, backup policies, and RGPD/GDPR compliance requirements.

===== Storage Tiers =====

NeuroSpin uses a three-tier storage model:

^ Tier ^ Path ^ Speed ^ Backed Up ^ Purpose ^ Quota ^
| Home | /home/$USER | Medium | Yes (daily) | Personal scripts, configs, small files | 100 GB |
| Project | /data/projects/$PROJECT | Fast (NVMe) | Yes (weekly) | Active analysis data, derivatives | Negotiated per project |
| Archive | /archive/$PROJECT | Slow (tape+disk) | Yes (triple) | Raw data, DICOM, final results | Unlimited (request-based) |
| Scratch | /scratch/$USER | Very fast (NVMe) | **NO** | Temporary computation files | 2 TB (auto-purged after 30 days) |

**Important**: `/scratch` is **not backed up** and is purged automatically every 30 days. Never store important data there. Always copy results to `/data/projects/` or `/archive/` before the purge deadline.

===== DICOM Archive =====

All MRI DICOM data acquired at NeuroSpin is automatically pushed to the NeuroSpin DICOM archive at acquisition time. Access is through the `ns-fetch` tool:

<code bash>
# List all sessions for a subject
ns-fetch --list --subject NS123456

# Download DICOMs for a specific session
ns-fetch --subject NS123456 --session 001 --output /data/projects/my_project/dicoms/

# Download only a specific sequence (e.g., T1w)
ns-fetch --subject NS123456 --session 001 --sequence T1w \
         --output /data/projects/my_project/dicoms/
</code>

The DICOM archive web interface is at http://archive.neurospin.fr/ (intranet only).

===== BIDS Format =====

All data at NeuroSpin should be organised in **BIDS format** (Brain Imaging Data Structure, https://bids-specification.readthedocs.io/). This is mandatory for data entering the central archive.

Example BIDS directory structure:
<code>
/data/projects/my_project/
├── dataset_description.json
├── participants.tsv
├── README
└── sub-01/
    └── ses-01/
        ├── anat/
        │   └── sub-01_ses-01_T1w.nii.gz
        ├── func/
        │   ├── sub-01_ses-01_task-myexperiment_bold.nii.gz
        │   └── sub-01_ses-01_task-myexperiment_bold.json
        └── fmap/
            ├── sub-01_ses-01_magnitude.nii.gz
            └── sub-01_ses-01_phasediff.nii.gz
</code>

Use `heudiconv` or `dcm2bids` to convert DICOMs to BIDS format:

<code bash>
# Using heudiconv (recommended at NeuroSpin)
heudiconv -d /data/projects/my_project/dicoms/NS{subject}/ses-{session}/ \
          -s 01 -ss 01 \
          -f /data/projects/my_project/code/heuristic.py \
          -o /data/projects/my_project/bids/ \
          -c dcm2niix --bids --overwrite
</code>

BIDS validator: `bids-validator /data/projects/my_project/bids/` (available after `module load bids-validator`).

===== Project Setup Procedure =====

Before starting a new project:

  - Email data-mgmt@neurospin.fr to register your project and get a project directory allocated.
  - Provide: project name, PI name, expected number of subjects, expected data volume, estimated project end date.
  - You will receive: project path (`/data/projects/your_project_name`), storage quota, and a project ID (e.g., `NS-2024-042`).

===== Data Naming Conventions =====

Use consistent, BIDScompliant subject IDs. NeuroSpin subject IDs follow the format:
`NS[6-digit-number]` (e.g., NS123456). These are generated by the MRI booking system.

Do NOT use participant names or initials in file names or directory names — this is a RGPD violation.

===== RGPD / GDPR Compliance =====

NeuroSpin data handling is governed by RGPD (EU General Data Protection Regulation).

Key rules:
  * **Never** store identifiable data (name, date of birth, contact info) outside the DICOM archive or the dedicated secure participant database.
  * Raw DICOMs are identifiable (they contain participant metadata in headers). They must stay in `/archive/` or the DICOM archive — never copy them to `/scratch/`.
  * Anonymise (using `pydeface` for T1 images, or `heudiconv` with defacing) before sharing or publishing data.
  * Data transfer outside NeuroSpin (e.g., to a collaborator or to an HPC cluster like Jean Zay) requires a signed data transfer agreement (DTA). Contact data-mgmt@neurospin.fr.
  * For Jean Zay specifically: submit a DTA request to IDRIS (idris.fr/jean-zay/cpu/jean-zay-cpu-hw.html) and to CEA legal.

<code bash>
# Deface a T1 image before sharing (removes facial structure)
pydeface sub-01_T1w.nii.gz --outfile sub-01_T1w_defaced.nii.gz
</code>

===== Backup Policy =====

^ Location ^ Backup frequency ^ Retention ^ Recovery time ^
| /home/$USER | Daily snapshot | 30 snapshots | < 4 hours |
| /data/projects/ | Weekly full + daily incremental | 1 year | < 24 hours |
| /archive/ | Triple-redundant (2× disk + 1× tape) | Indefinite | 1–5 days |
| /scratch/ | **Not backed up** | N/A | N/A |

To restore a lost file, email it-support@neurospin.fr with the full path and the approximate date of loss.

===== Quotas and Requesting More Storage =====

To check your current usage:
<code bash>
# Home directory
du -sh /home/$USER

# Project directory
du -sh /data/projects/my_project/

# Full quota status
ns-quota --user $USER
</code>

To request a quota increase, email data-mgmt@neurospin.fr with justification (data type, expected growth over 12 months). Requests are reviewed weekly.
